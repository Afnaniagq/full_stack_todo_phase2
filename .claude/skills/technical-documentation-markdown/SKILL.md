# Technical Documentation (Markdown)

## Instructions

1. **Document structure**
   - Start with a clear title and summary
   - Organize content using headings and subheadings
   - Use consistent section ordering (Overview, Setup, Usage, Examples)

2. **Content formatting**
   - Use lists for steps and requirements
   - Highlight commands and code with fenced blocks
   - Emphasize key terms with bold and italics

3. **Code & examples**
   - Include concise, runnable examples
   - Specify languages in code fences
   - Keep examples focused on a single concept

4. **Maintenance & clarity**
   - Keep documentation up to date with the codebase
   - Write concise, actionable instructions
   - Link to related resources and references

## Best Practices
- Write for your target audience
- Keep sentences short and direct
- Use consistent terminology
- Prefer examples over long explanations
- Avoid unnecessary formatting
- Review docs as part of code reviews
